‚úÖ Summary / Rules of Thumb

  2. External files (outside website): Use Absolute Paths

2. HTML Boilerplate Code

üîπ First Principle

  • Browser needs predictable instructions.
  • Can‚Äôt guess ‚Üí needs info like version, encoding, title, responsiveness.

üîπ Core Problem

  • Without setup info:
    • Browser won‚Äôt know HTML version, encoding, title, or viewport behavior.

üîπ Solution ‚Üí Boilerplate Template

                <html lang="en">
                    
                <head>
                        
                    <meta charset="UTF-8">
                        
                    <meta name="viewport" content="width=device-width, initial-scale=1.0">
                        
                    <title>Document</title>
                    
                </head>
                
                <body>

                    
                </body>

üîπExplanation Of Boilerplate Code

  1. <!DOCTYPE html>

    • Tells browser to use HTML5 standards mode.
    • Must be first line.

  2. <html lang="en">

    • Root element.
    • lang = content language (important for SEO + accessibility).

  3. <head>....</head>

    • Holds metadata (not visible content).

  4. <meta charset="UTF-8">

    • Defines encoding.
    • UTF-8 supports nearly all characters.

  5. <meta name="viewport" ......>

    • Enables responsive design.
    • width=device-width ‚Üí fit device width.
    • initial-scale=1.0 ‚Üí zoom 100%.

  6. <title>Document</title>

    • Text in tab, bookmarks, search results.

  7. <body>.....</body>

    • Holds visible content (headings, text, images, links, etc.).

üîπ Why Website Works Without Boilerplate?

Browsers = forgiving. They auto-correct:

  1. Missing DOCTYPE ‚Üí Quirks Mode.
  2. Missing / ‚Üí auto-generated in DOM.
  3. Missing charset ‚Üí browser guesses.
  4. Missing title ‚Üí filename used.

‚ö†Ô∏è Why Still Use Boilerplate?

  • Unpredictable Rendering: Different quirks in Chrome/Firefox/Safari.
  • Broken Characters: Special symbols may fail.
  • Bad Mobile UX: Without viewport tag ‚Üí tiny, unreadable.
  • Poor SEO & Accessibility: Search engines + screen readers suffer.

üëâ Boilerplate = professional, reliable, future-proof foundation.

üîπ Fundamental Truth

üîπ Logical Solution

  • No inherent meaning.
  • Useful for applying styles to a whole block.

‚úÖ Example:

5. class & id (Labels for Boxes)

  • Solution:

    • class ‚Üí reusable (many elements).

‚úÖ Example:

                    <div class="profile-card">...</div>

‚úÖ CSS targeting:

  • #main-header { ... } ‚Üí one element.
  • .profile-card { ... } ‚Üí all with class.
  • #featured-profile { ... } ‚Üí one special case.
