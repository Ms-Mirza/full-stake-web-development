üìíHow Does Internet Work?


Principle 1: The Need to Share Information

  • Goal: Transfer data from Computer A ‚Üí Computer B (not in same room).
  • Primitive solution: Use USB stick ‚Üí slow, not scalable.
  • Better solution: Run a cable (ethernet/fiber) ‚Üí send electrical signals/light pulses.
    • High voltage = 1
    • Low voltage = 0
  • This forms a direct link ‚Üí able to send bits (1s, 0s).

Principle 2: The Problem of Scale

  • With 3 computers (A, B, C) ‚Üí need many cables.
  • For 100 computers ‚Üí need 4,950 cables ‚Üí not scalable.
  • Solution: Use a Switch (central device).
    • Each computer connects to switch.
    • Switch forwards message only to correct destination.
  • This creates a Local Network (LAN).

Principle 3: The Problem of Connecting Networks

  • Each office may have its own LAN.
  • How to connect LANs? Cannot wire every computer to every other.
  • Solution: Use a Router (special device).
    • Router connects multiple networks.
    • Routes traffic between networks.
  • The Internet = a network of networks (millions of LANs connected via routers).

Principle 4: The Problem of Finding Anyone (Addressing)

  • Need a unique address for every device in global network.
  • IP Address introduced:
    • Works like a postal address.
    • Routers use destination IP to forward packets toward the right path (hop by hop).

Principle 5: Reliability & Fairness

  • Sending a large file all at once = problems:
    1. Blocks other users (monopoly).
    2. Error in 1 bit ‚Üí file ruined.
  • Solution: Break data into Packets.
    • Each packet contains:
      • Payload (data piece).
      • Destination IP.
      • Sender IP.
      • Sequence number (to reassemble).
    • Packets may take different routes ‚Üí receiver reassembles.
  • This is handled by TCP/IP protocol.

üîë Summary from First Principles

  1. Share data ‚Üí Direct Link.
  2. Many computers ‚Üí Local Network with Switch.
  3. Many networks ‚Üí Internet via Routers.
  4. Find devices ‚Üí IP Address system.
  5. Reliability ‚Üí TCP/IP with Packets.

IP Address

  1. Public IP Address

    • Global, unique address (like company‚Äôs main phone).
    • Must be unique worldwide.
    • Assigned by ISP.
    • Example: Router‚Äôs IP (seen on whatismyip.com).
    • Servers like Google/Netflix have public IPs.

  2. Private IP Address

    • Used within local networks (e.g., home Wi-Fi).
    • Like employee‚Äôs internal extension number.
    • Unique only in local scope.
    • Assigned by Router.
    • Reserved ranges:
      • 10.0.0.0 ‚Äì 10.255.255.255 (large networks)
      • 172.16.0.0 ‚Äì 172.31.255.255 (medium networks)
      • 192.168.0.0 ‚Äì 192.168.255.255 (home networks)

MAC Address

Permanent, hardware ID for network devices.

48-bit number ‚Üí written as 12 hex digits.

Formats:

  • Colon-separated: 3C:22:FB:A3:B4:C5
  • Hyphen-separated: 3C-22-FB-A3-B4-C5
  • Cisco style: 3c22.fba3.b4c5
  • No separator: 3C22FBA3B4C5

Find MAC Address:

  • Windows: ipconfig /all or getmac /v
  • macOS/Linux: ifconfig

Port Number

Identifies specific apps/services on a device.

Works with IP to route traffic.

16-bit unsigned integer (0‚Äì65535).

Categories of Ports

Well-Known Ports (0‚Äì1023)

  • Controlled by IANA.
  • Require admin rights.
  • Examples: 21 (FTP), 22 (SSH), 25 (SMTP), 53 (DNS), 80 (HTTP), 443 (HTTPS)

Registered Ports (1024‚Äì49151)

  • For applications, less strict.
  • No admin rights needed.
  • Examples: 3000 (React/Node dev server), 3306 (MySQL), 5432 (PostgreSQL), 27017 (MongoDB), 8080 (alt web server)

Dynamic / Ephemeral Ports (49152‚Äì65535)

  • Temporary client-side ports.
  • Auto-assigned by OS.
  • Used for outgoing connections.

DNS (Domain Name System)

Translates domain names ‚Üí IP addresses.

Without DNS, we‚Äôd memorize IPs.

Process of DNS Lookup

Check Cache (fastest option):

  • Browser cache ‚Üí OS cache ‚Üí Router cache.
  • If found, use cached IP.

Recursive Resolver (asks on behalf):

  • Usually ISP‚Äôs server (or public: 8.8.8.8, 1.1.1.1).
  • Does the "hard work".

Hierarchical Search:

  • Root Server: Points to TLD servers (.com, .org, etc.)
  • TLD Server: Points to authoritative servers for domain
  • Authoritative Server: Gives actual IP (source of truth)

Journey Back:

  • IP returned ‚Üí cached by resolver ‚Üí sent to OS ‚Üí given to browser

Final Connection:

Browser connects to IP (e.g., 142.250.72.206) on port 443 and requests webpage.
